Background story: every time i approach potential client after manually scraping their phone number i have to repeatedly chat multiple number with same template (only change certain part to suit the client) and i realize it can be more efficient if i create CRM Dashboard

I want to create simple CRM dashboard with the purpose:
1. It have a table with necessary column like phone number, client name, client organization, last contact and action button that integrate with webwhatsapp.js
2. The action button will give a pop up with the interface of the chat history, drop down of template chat and input for custom answer chat. So i dont have to open my Whatsapp to read and send the chat for client
3. Whatsapp integration only accept the text for the number in whitelist. The whitelist is updated everytime I manually add in CRM table row
4. Create a pgsql with ORM prisma for DB. and for FE choose vite and for BE is up to claude which one that run quick and efficient
5. DB save all the client data, chat history and any other stuff that might add in this project
